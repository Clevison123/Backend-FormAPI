# FormAPI

## Project Overview

FormAPI is a REST API built with ASP.NET Core to manage user registrations through a simple form.

The application allows users to be created, retrieved, updated, and deleted using standard CRUD operations.

---

## Features

Currently, the API supports:

* Creating users.
* Retrieving all registered users.
* Retrieving a user by ID.
* Updating user information.
* Deleting users from the system.

---

## Data Structure

Each user contains the following fields:

* Name
* Email
* Phone Number
* Address

The ID field is automatically generated by the database and is not provided by the user.

---

## Implemented Validations

Some validations are performed during user creation and update operations.

### Duplicate Email Validation

Before creating a new user, the system checks whether another user is already registered with the same email address.

If the email is already in use, the registration is blocked.

### User Existence Validation

During retrieval, update, and delete operations, the system verifies whether the requested user exists in the database.

If the user cannot be found, an exception is thrown.

---

## Project Architecture

The project follows a separation of responsibilities approach.

### Controllers

Responsible for handling HTTP requests and responses.

### Services

Responsible for business logic and database interactions.

### DTOs (Data Transfer Objects)

Responsible for controlling which data enters and leaves the API.

### Entity

Represents the database table structure.

### Data

Responsible for Entity Framework configuration and database connection management.

### Migrations

Responsible for creating and evolving the database schema.
